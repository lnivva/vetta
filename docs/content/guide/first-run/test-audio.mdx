# Generate Test Audio

To verify the pipeline's accuracy—specifically speaker diarization and financial entity normalization—you need a high-quality test file. The scripts in `dev_scripts/` simulate a multi-speaker earnings call with dense technical and financial terminology.

## 1. Prerequisites

Before running the generators, ensure your system has the necessary tools and voice models installed.

<Tabs>
    <Tab label="Ubuntu 24.04">
        We use **Piper**, a fast, local neural TTS engine, to provide high-quality "macOS-like" voices.

        ```bash
        # 1. Install media tools
        sudo apt update && sudo apt install -y ffmpeg wget tar

        cd ~

        # 2. Download Piper binary (installs to ~/piper/)
        wget https://github.com/rhasspy/piper/releases/download/v1.2.0/piper_amd64.tar.gz
        tar -xf piper_amd64.tar.gz

        # 3. Download voice models into ~/piper/
        cd ~/piper

        # Speaker 1 — Amy (female, CEO)
        wget -nc https://huggingface.co/rhasspy/piper-voices/resolve/main/en/en_US/amy/low/en_US-amy-low.onnx
        wget -nc https://huggingface.co/rhasspy/piper-voices/resolve/main/en/en_US/amy/low/en_US-amy-low.onnx.json

        # Speaker 2 — Lessac (male, CFO)
        wget -nc https://huggingface.co/rhasspy/piper-voices/resolve/main/en/en_US/lessac/medium/en_US-lessac-medium.onnx
        wget -nc https://huggingface.co/rhasspy/piper-voices/resolve/main/en/en_US/lessac/medium/en_US-lessac-medium.onnx.json

        # Speaker 3 — Kristin (female, Analyst)
        wget -nc https://huggingface.co/rhasspy/piper-voices/resolve/main/en/en_US/kristin/medium/en_US-kristin-medium.onnx
        wget -nc https://huggingface.co/rhasspy/piper-voices/resolve/main/en/en_US/kristin/medium/en_US-kristin-medium.onnx.json

        cd ~
        ```
    </Tab>
    <Tab label="macOS">
        ```bash
        # Install ffmpeg via Homebrew (say and high-quality voices are built-in)
        brew install ffmpeg
        ```
    </Tab>
</Tabs>

## 2. Generate Test Audio

The generator scripts live in the repository under **`dev_scripts/`**. Both output a **16 kHz mono WAV** file to `/tmp/test.wav`.

<Tabs>
    <Tab label="Ubuntu 24.04 (Linux)">
        This script uses **Piper** neural models to simulate a realistic, multi-speaker call.
        It references `~/piper/` for the binary and voice models, so it works from any working directory.

        ```bash

        # From the repo root
        chmod +x dev_scripts/gen_test_audio_linux.sh
        ./dev_scripts/gen_test_audio_linux.sh
        ```

        :::tip Source
        [`dev_scripts/gen_test_audio_linux.sh`](https://github.com/your-org/vetta/blob/main/dev_scripts/gen_test_audio_linux.sh)
        :::

        :::warning Permission error on re-run?
        If you see `rm: cannot remove '/tmp/stt_test/...': Permission denied`, a previous run
        (possibly via `sudo`) left root-owned temp files. Clear them first:
        ```bash
        sudo rm -rf /tmp/stt_test /tmp/test.wav
        ```
        :::
    </Tab>

    <Tab label="macOS">
        This script uses the native `say` command with high-quality neural voices.

        ```bash
        # From the repo root
        chmod +x dev_scripts/gen_test_audio_macos.sh
        ./dev_scripts/gen_test_audio_macos.sh
        ```

        :::tip Source
        [`dev_scripts/gen_test_audio_macos.sh`](https://github.com/your-org/vetta/blob/main/dev_scripts/gen_test_audio_macos.sh)
        :::
    </Tab>
</Tabs>

## 3. Expected Results

After processing `/tmp/test.wav` through the service, verify the output against these targets.

### Entity Normalization

The `TranscriptPostProcessor` should automatically transform raw Whisper outputs into professional financial text:

| Raw Whisper | Expected Correction |
| :--- | :--- |
| `mongo db` | **MongoDB** |
| `q four` | **Q4** |
| `c e o` / `c f o` | **CEO** / **CFO** |
| `non gaap` | **non-GAAP** |
| `opex` / `capex` | **OpEx** / **CapEx** |
| `a w s` | **AWS** |
| `open ai` / `chat gpt` | **OpenAI** / **ChatGPT** |

### Diarization

The pipeline should identify **3 distinct speakers**.

* **Speaker A:** First and Third segments (CEO — Strategy)
* **Speaker B:** Second segment (CFO — Financials)
* **Speaker C:** Final segment (Analyst — Question)
