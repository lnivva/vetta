# Generate Test Audio

<p>
    You need an audio file to test the pipeline. Pick the option that matches your platform. We provide helper scripts
    in the `_dev_script/` directory that generate realistic multi-speaker audio with acoustic fingerprinting for
    reliable diarization testing.
</p>

<Tabs>
    <Tab label="macOS">

        The macOS script uses the built-in `say` command and `ffmpeg` to generate speech, apply acoustic profiles
        (pitch, tempo, EQ), and combine the segments with silence padding.

        **Prerequisites**: Ensure you have `ffmpeg` installed (e.g., `brew install ffmpeg`).

        ```bash
        # Run the generator script
        ./_dev_script/gen_test_audio_macos.sh
        ```

        This will output a perfectly formatted 16kHz, mono, PCM-16 file at `/tmp/test.wav` featuring three distinct
        speakers (CEO, CFO, Analyst).

    </Tab>
    <Tab label="Linux">

        The Linux script uses the Piper TTS engine to generate high-quality speech. You must first run the setup script
        to download the Piper binary and necessary ONNX voice models.

        **Prerequisites**: Ensure you have `curl`, `tar`, and `ffmpeg` installed (e.g., `sudo apt-get install -y curl
        tar ffmpeg`).

        ```bash
        # 1. Download and configure Piper + voices to ~/piper
        ./_dev_script/setup_piper_linux.sh

        # 2. Run the generator script
        ./_dev_script/gen_test_audio_linux.sh
        ```

        This will output a perfectly formatted 16kHz, mono, PCM-16 file at `/tmp/test.wav` featuring three distinct
        speakers (CEO, CFO, Analyst).

        *Alternative: Use an existing audio file*

        ```bash
        cp /path/to/your/earnings-call.mp3 /tmp/test.mp3
        ```

    </Tab>
</Tabs>

:::tip

WAV is the preferred format because Whisper and PyAudio handle it natively without any conversion.

Other formats (MP3, AIFF, etc.) are supported through ffmpeg but require an internal conversion step that adds processing time.

:::
