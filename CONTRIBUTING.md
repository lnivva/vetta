# Contributing

Thanks for your interest in contributing! 🎉

## Getting Started

1. Fork the repository
2. Clone your fork:

   ```bash
   git clone https://github.com/your-username/your-repo.git
   ```
3. Create a branch for your changes:

   ```bash
   git checkout -b feature/your-feature-name
   ```

## Development

* Make your changes in small, focused commits
* Write clear commit messages
* Add or update tests if applicable
* Ensure the project builds and tests pass locally

## Code Style

* Follow the existing code style and conventions
* Keep code readable and simple
* Run any linters or formatters before submitting

## Submitting Changes

1. Push your branch:

   ```bash
   git push origin feature/your-feature-name
   ```
2. Open a Pull Request (PR)

## Pull Request Guidelines

* Clearly describe what your PR does and why
* Link related issues if applicable
* Keep PRs small and focused
* Be open to feedback and make changes if requested

## Reporting Issues

If you find a bug or have a suggestion:

* Check if it already exists in the issues
* If not, open a new issue with:

  * A clear title
  * Description of the problem
  * Steps to reproduce (if applicable)

## Code of Conduct

This project follows the guidelines outlined in [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.
